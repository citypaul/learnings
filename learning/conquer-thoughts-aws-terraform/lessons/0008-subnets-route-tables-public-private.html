Subnets, Route Tables, and Public versus Private

Lesson 0008 - Conquer Thoughts AWS/Terraform - 2026-06-14

Part of your mission to build a durable mental model of the Conquer Thoughts production walking skeleton so you can explain, debug, and safely extend the AWS/Terraform/GitHub Actions rail.

After this lesson you'll be able to classify a subnet as public or private by reading its route table, not by guessing from its name.

Concrete example: the public task subnets have an internet route

The network module creates an internet gateway and a public route table. The key route is: 

destination_cidr_block = "0.0.0.0/0"
gateway_id             = aws_internet_gateway.this.id

That route says "for IPv4 destinations outside the VPC, send traffic to the internet gateway." The module associates this route table with the public subnet IDs. The public subnets also have map_public_ip_on_launch = true, and the ECS service explicitly assigns public IPs for the BFF tasks. This gives the day-one Fargate service a simple egress path for image pulls and runtime calls without first introducing NAT gateways or VPC endpoints.

Concrete example: the private origin subnets do not have that route

The private route table in the module has no explicit 0.0.0.0/0 route to the internet gateway. It only has the implicit local VPC route that AWS provides. The internal ALB is placed in those private origin subnets, so the ALB has VPC-local reachability but no direct public internet path.

AWS's subnet docs define the practical rule: if a subnet's route table has a route to an internet gateway, it is a public subnet; if it does not, it is a private subnet (AWS subnet docs). The internet gateway docs add the subtle bit: a subnet without an internet-gateway route cannot communicate directly with the internet, even if an instance in it has a public IPv4 address (AWS internet gateway docs).

The principle

"Public" and "private" are routing outcomes. They are not special AWS subnet types, and they are not created by a name tag. A route table is a list of destination CIDR blocks and targets; AWS uses the most specific matching route to decide where traffic goes (AWS route table docs).

Security groups are the next gate

Routing answers "where can packets go?" Security groups answer "which packets are allowed into or out of this resource?" AWS security groups are stateful resource-level firewalls (AWS security group docs). In this project, route tables make the ALB private, and security groups then make the ALB accept HTTP only from the CloudFront VPC origin service-managed security group.

QuestionProject answer
What makes the task subnets public? Their route table has 0.0.0.0/0 -> internet gateway.
What makes the origin subnets private? Their route table has no direct internet-gateway route.
What protects the internal ALB after routing? The ALB security group only allows inbound traffic from CloudFront's VPC origin service-managed security group.

Check yourself

  1. What is the single fastest way to tell whether an AWS subnet is public?
    Answer

    Inspect the subnet's associated route table for a route to an internet gateway, usually 0.0.0.0/0 -> igw-....

  2. Does map_public_ip_on_launch = true alone make a subnet public?
    Answer

    No. Public IP assignment matters, but the route table still needs an internet-gateway route for direct internet communication.

  3. Why are the BFF tasks placed in public subnets in the current walking skeleton?
    Answer

    It gives the day-one Fargate tasks simple outbound internet access without NAT gateways or VPC endpoints.

  4. Why is the ALB placed in private origin subnets?
    Answer

    The public entry point is CloudFront. The ALB should be reachable through the CloudFront VPC origin path, not directly from the internet.

  5. If CloudFront cannot reach the internal ALB, what are the first network checks?
    Answer

    Check the CloudFront VPC origin status, available private subnet IPs, the ALB security-group ingress from CloudFront-VPCOrigins-Service-SG, and the CloudFront distribution's origin behavior.
