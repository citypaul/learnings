CloudFront VPC Origins

Lesson 0009 - Conquer Thoughts AWS/Terraform - 2026-06-14

Part of your mission to build a durable mental model of the Conquer Thoughts production walking skeleton so you can explain, debug, and safely extend the AWS/Terraform/GitHub Actions rail.

After this lesson you'll be able to explain how CloudFront reaches the internal ALB without making the ALB public.

Concrete example: CloudFront is the only public front door

The production architecture uses one CloudFront distribution for both static web assets and API routes. The default origin is the private S3 assets bucket through Origin Access Control. The API routes use a CloudFront VPC origin that points at the internal ALB.

AWS describes VPC origins as a way to deliver content from origins inside a VPC, such as Application Load Balancers, Network Load Balancers, and EC2 instances in private subnets, while keeping CloudFront as the single public entry point (AWS CloudFront VPC origins docs).

Concrete example: Terraform creates a private bridge

The static-edge module has four important pieces:

Terraform pieceWhat it does
aws_cloudfront_vpc_origin.alb Registers the internal ALB as a CloudFront VPC origin.
data.aws_security_group.cloudfront_vpc_origin Finds the CloudFront-created service-managed security group named CloudFront-VPCOrigins-Service-SG.
aws_vpc_security_group_ingress_rule.cloudfront_to_alb Allows inbound HTTP from that service-managed security group to the ALB security group.
aws_cloudfront_distribution.this Routes /api/* and /health to the VPC origin; all other paths go to S3.

The ordering matters. The service-managed security group exists only after the VPC origin is created, so the data source depends on the VPC origin. Then the ALB ingress rule depends on that data source. Terraform is not just creating boxes here; it is encoding a dependency chain for a managed AWS networking feature.

The principle

A CloudFront VPC origin is not a public ALB with a secret header. It is a CloudFront-managed private path into your VPC. The ALB remains internal. The ALB security group allows traffic from CloudFront's service-managed security group. CloudFront then decides, by cache behavior, which URLs use that private API origin.

Subtle prerequisites and limits

AWS requires the VPC that contains the origin to have an internet gateway, but AWS also says that this internet gateway is not used for routing traffic to origins inside the subnet. The origin subnet must have available IPv4 addresses so CloudFront can create service-managed network interfaces. AWS also says the service-managed security group should not be edited directly; the origin's security group is where you allow traffic from it.

There are feature limits to remember. AWS's VPC origin docs list unsupported features, including gRPC traffic and Lambda@Edge origin triggers for VPC origins. The project does not need those features for the walking skeleton, but they matter before treating VPC origins as a drop-in replacement for every CloudFront origin pattern.

Check yourself

  1. What problem does a CloudFront VPC origin solve for this project?
    Answer

    It lets CloudFront route API requests to an internal ALB without making the ALB directly public.

  2. Which security group should the project edit to permit CloudFront traffic?
    Answer

    The ALB security group should allow inbound traffic from the CloudFront VPC origins service-managed security group. The service-managed security group itself should not be edited directly.

  3. Why does the Terraform data source for CloudFront-VPCOrigins-Service-SG depend on the VPC origin?
    Answer

    Because CloudFront creates that service-managed security group as part of the VPC origin setup. Looking it up too early can fail.

  4. Does the VPC internet gateway carry CloudFront VPC origin traffic to the internal ALB?
    Answer

    No. AWS requires the VPC to have an internet gateway, but says it is not used for routing traffic to origins inside the subnet.

  5. Which origin protects static assets: the VPC origin or Origin Access Control?
    Answer

    Origin Access Control protects the private S3 assets bucket. The VPC origin is for the internal ALB/API path.
